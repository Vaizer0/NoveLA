# Анализ 27 нативных источников для миграции в Lua

Этот отчет содержит технические детали и специфические требования для каждого источника, выявленные в ходе анализа.

## Группа 1: Простые (HTML + GET)
*Эти источники используют стандартный парсинг и не требуют сложных механизмов.*

1.  **AllNovel**: Использует `PAGE_BASED` пагинацию глав. Требуется поддержка `transformCoverUrl` (стиль NovelBin).
2.  **BacaLightnovel**: Простой, индонезийский язык.
3.  **Bookhamster**: Простой, русский язык.
4.  **NoBadNovel**: Требует поддержку сложных CSS селекторов с экранированием (например, `sm\:text-lg`).
5.  **NovelFull**: Аналогично AllNovel, использует `PAGE_BASED` пагинацию.
6.  **NovelHall**: Простой, нет обложек в каталоге.
7.  **Novelku**: Аналогично BacaLightnovel.
8.  **RoyalRoad**: Простой, хорошая структура.
9.  **WuxiaWorld_site**: Использует AJAX для списка глав (POST запрос).

## Группа 2: AJAX и Динамический контент
*Требуют выполнения дополнительных запросов для получения списка глав или поиска.*

10. **Jaomix**: Список глав загружается через AJAX (POST запросы) постранично в обратном порядке. Нужна задержка между запросами (`delay`).
11. **NovLove**: Список глав через AJAX (`/ajax/chapter-archive?novelId=...`).
12. **NovelBin**: novelId извлекается из мета-тегов, список глав через AJAX.
13. **NovelBuddy**: bookId извлекается регулярным выражением из скрипта, список глав через AJAX API.
14. **ReadNovelFull**: novelId из атрибута `data-novel-id`, список глав через AJAX.
15. **ScribbleHub**: seriesId из URL, список глав через AJAX (POST запрос).
16. **Ttkan**: novelId из URL, список глав через JSON API, требует ручного парсинга JSON или регулярных выражений.
17. **Twkan**: bookId из URL, список глав через AJAX `/ajax_novels/chapterlist/`.

## Группа 3: Кодировки и Китайские источники
*Требуют поддержки GBK и специфических манипуляций со строками.*

18. **PiaoTia**: Кодировка **GBK**. Список глав через AJAX. Кастомная обработка `document.write` в тексте глав.
19. **Quanben5**: Кастомный поиск через JSONP. Требует `encodeURI` (реализован как `url_encode`), кастомный Base64 с алфавитом и `unescape_unicode`.
20. **Shuba69**: Кодировка **GBK**. POST поиск с кодировкой GBK. Обратный порядок глав.
21. **Novel543**: Главы разбиты на подстраницы, требуется рекурсивная склейка текста.

## Группа 4: Особые требования и Защита
*Сложная логика, защита контента или настройки.*

22. **WtrLabSource**: Требует `ScreenConfig` (выбор режима AI/Raw и языка). Использует AES дешифровку контента. Требует `get_preference`/`set_preference`.
23. **RanobeHub**: (Уже мигрирован) — использует JSON API и тома.
24. **RanobeLib**: Очень сложный источник. JSON API, сложная структура томов, защита от ботов, кастомные заголовки.
25. **Ifreedom**: Простой, но требует очистки мусора в тексте.
26. **NovelFire**: Использует `PAGE_BASED` пагинацию для глав.
27. **LocalSource**: Не требует миграции (внутренний).

## Общие выводы для движка Lua:
1.  **Интерфейс**: Нужна поддержка `volume` в списке глав (добавлено).
2.  **HTTP**: Поддержка `charset` и `headers` обязательна (добавлено).
3.  **Утилиты**: `aes_decrypt`, `url_resolve`, `unescape_unicode`, `regex_match` (добавлено).
4.  **UI**: Поддержка рендеринга настроек через `getScreenConfig` (добавлено).
